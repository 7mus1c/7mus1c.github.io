http 请求的时候就是一进一出，客户端发生请求到服务端，服务端接到请求，发送响应到客户端。

## 当我们输入网址后发生了什么

1. 输入网址并回车
2. 解析域名
3. 浏览器发送 HTTP 请求
4. 服务器处理请求
5. 服务器返回 HTML 响应
6. 浏览器处理 HTML 页面
7. 继续请求其他资源

### 详细过程

- 用户输入网址
- 跨越网段 （从局域网跳到互联网，中间可能是防火墙、网关、代理等）
- DNS 服务器域名解析 （浏览器把域名转成 IP 地址）
- 交换设备中转 （很多路由设备）
- 数据包在网络中传输
- 到达到服务器反向代理转发
- 请求达到服务器
- 服务器接到请求给予相应
- 数据包在网络中传输 （与来时的路未必一样）
- 到达客户端浏览器渲染
- 断开连接 （如果有其他请求继续）

## HTTP 协议

HTTP 是超文本协议，因为传输的不仅仅是文本，还有可能是图片、css、js、音视频文件等。  
HTTP 协议是从客户端到服务器的请求和从服务器到客户端的相应的一种约束和规范。  
HTTP 的发展 1991 年的 0.9 版本至 1996 年的 1.0 版本再到 1999 年的 1.1 版本到最后 2015 年 2 的版本。现在市面常用的 HTTP 协议是 1.1 的版本。  
HTTP3 对于 1 和 2 是一种全新的协议，更换了底层的协议。 1 和 2 是 **TCP** 协议，而 3 是 **UDP** 协议。UDP 能解决很多性能上的问题，和传输效率的问题。
HTTP 默认端口号是：80，HTTPS 的默认端口号：443。

ISO 协议是标准组织定义的协议，试试协议是一开始逐步发展起来的协议，但是并不完善。后来发现 TCP/IP 设计的比较粗糙。  
随着软件复杂度的提升发现一个应用层是不够的，掺杂了大量的东西导致结构混乱，网络接口层内部还包含数据链路层并不合理。所以才重新指定了 ISO 协议来完善。

## TCP/IP 协议

1. 应用层
   - 为用户提供服务，如 HTTP 协议、FTP 协议、SSH 协议、DNS 协议、SMTP 协议等
2. 传输层
   - 给应用层实体提供端到端的通信，保证数据的传送及完整
   - 该层主要有两个协议：传输控制协议（TCP）、用户数据协议（UDP）
3. 网络层
   - 主要解决主机到主机的通信。
4. 网络接口层
   - 负责监视数据在主机和网络之间的交换
   - 其中还包含了数据链路层

## ISO 协议

1. 应用层
   - 给软件提供服务：HTTP 协议、FTP 协议、SHH 等
2. 表示层
   - 用于处理在两个通信系统中交换信息的表示方式，主要包括数据格式变换、加密与解密、压缩与恢复等功能
3. 会话层
   - 负责两个点之间的传输连接，确保点到点传输不中断，管理数据交换等功能。比如封装 HTTPS，密钥交换等
4. 传输层
   - 传输层有两个重要的协议，TCP 和 UDP
   - TCP 是面向连接的协议，UDP 是无连接的协议
5. 网络层
   - IP 层，IP 地址就是在网络层分配，比如 ping 一个机器时，那时的包叫 ICMP 协议
   - ICMP 协议也在网络层上，但是这一层最重要的还是 IP 协议
6. 数据链路层
   - 涉及网卡（MAC 地址）
7. 物理层
   - 实际上指的是硬件、网线等等硬件设备

## TCP/IP 协议工作过程

请求数据，请求参数通过应用层塞到 HTTP 协议里面进行封装，然后传输层通过 TCP 协议再封装 HTTP，封装好以后再传到网络层封装成原始数据报，再往下传输然后光/电信号。  
然后就是通过电联或者光缆传出去，另外一端收到光/电信号进行转化，转化城数字信号，然后串行传输协议，再转成原始数据报，再解开 TCP、HTTP，HTTP 解开以后服务器就可以使用了。

## HTTP 工作过程

一次 HTTP 操作称之为一个事务（事务：当一次工作时，并不能一下子就完成，要分成若干步骤，每个步骤依赖上一个步骤，当然也可以没有依赖性，但是再执行当中若有一个步骤失败了，整个流程都失算失败了。事务就需要回滚，所有的操作都恢复成原始状态，如果操作数据的话，那么数据也要恢复成原始数据。），工作可以分为四步：

1. 客户机与服务器建立连接，创建 HTTP 连接，经过三次握手，然后创建 TCP 链路这样才可以发送请求
2. 客户机发送 HTTP 请求，格式：URL、协议版本号、MIME 信息（基础信息，头信息，和发送的数据等）
3. 服务器相应，格式：状态行（成功或错误的代码）、协议版本号、MIME 信息（基础信息、头信息、返回的数据等）
4. 客户端把数据进行渲染，断开连接。

**这里的事务回滚其实就是建立连接，获取内存，释放内存，断开连接**

## 请求与响应

- HTTP 请求组成：请求行、消息报头、请求正文（get 请求没有请求正文）
- HTTP 响应组成：状态行、消息报头、相应正文
- 请求行组成：以一个方法符号开头，后面跟着请求的 URI 和协议的版本号
- 状态行组成：服务器 HTTP 协议的版本号，服务器返回的响应状态码和状态码的文本描述
